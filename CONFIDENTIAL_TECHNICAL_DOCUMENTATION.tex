\chapter{Product Overview}
\label{ch:overview}

\section{Introduction}

\textbf{DeadlineIQ} is an ultra-premium, AI-driven productivity companion engineered to defeat procrastination and optimise cognitive focus. Unlike conventional productivity tools that rely on passive push alerts — which users habitually ignore — DeadlineIQ introduces a \textit{proactive, spatial, and cognitively aware} approach to task management.

The system analyses behavioural patterns in real time, predicts task-completion risks using a client-side machine learning model, and orchestrates personalised focus windows directly around the user's existing calendar commitments.

\begin{tcolorbox}[infobox, title={Core Value Proposition}]
DeadlineIQ replaces reactive reminders with \textbf{predictive intelligence} — it intervenes before deadlines are missed, not after. Every feature is designed to reduce cognitive load, not add to it.
\end{tcolorbox}

\section{Key Capabilities}

\begin{enumerate}[leftmargin=*, label=\textbf{\arabic*.}, itemsep=6pt]
  \item \textbf{Local Neural Network Risk Scoring} — A client-side Multi-Layer Perceptron (MLP) computes personalised procrastination probability scores without transmitting sensitive data to external servers.
  \item \textbf{Natural Language Task Parsing} — Powered by \textit{Gemini 2.5 Flash}, users dictate chaotic schedules in plain English and receive structured task objects with cognitive-load estimates and confidence scores.
  \item \textbf{Calendar-Aware Focus Scheduling} — Deep integration with the Google Calendar API allocates focus windows around existing commitments and peak-productivity periods.
  \item \textbf{Procrastination Forensics} — Recalculates behavioural delays to identify and categorise procrastination root causes (e.g., \textit{Fear of Failure}, \textit{Task Ambiguity}) with SVG trend charts.
  \item \textbf{Agentic Co-pilot (IQ Coach)} — A sliding sidebar AI agent executes scheduling actions, reschedules tasks, and commits all updates to Firestore in real time.
  \item \textbf{Habit Streak Tracking} — Glowing circular gauges monitor routine completions with Gemini-powered success probability forecasts.
  \item \textbf{Chrome Extension Integration} — Synchronises browser research directly to the cloud task dashboard without context-switching.
\end{enumerate}

\section{Firestore Security Rules}
\label{sec:rules}

All Firestore reads and writes are protected by production-grade security rules that enforce strict authenticated owner-only access:

\begin{tcolorbox}[codebox]
\begin{lstlisting}[numbers=none]
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write:
        if request.auth != null
        && request.auth.uid == userId;
    }
  }
}
\end{lstlisting}
\end{tcolorbox}

These rules guarantee that:
\begin{itemize}[noitemsep]
  \item Unauthenticated requests are rejected at the database layer.
  \item Each user can only access documents within their own \texttt{/users/\{userId\}/} namespace.
  \item Cross-user data leakage is architecturally impossible.
\end{itemize}

\chapter{Security Considerations}
\label{ch:security}

\section{Authentication Layer}
Firebase Authentication with Google OAuth 2.0 ensures every session is cryptographically signed and token-validated. Token expiry and refresh cycles are handled automatically by the Firebase SDK.

\section{Data Isolation at the Database Layer}
The Firestore namespace structure (\texttt{/users/\{userId\}/...}) combined with the security rules described in Section~\ref{sec:rules} makes cross-user data access architecturally impossible — not merely a policy decision.

\section{Client-Side ML Privacy}
The MLP neural network runs entirely within the browser's JavaScript engine. Behavioural training data and learned weights are persisted only to \texttt{localStorage} on the user's device.

\section{API Key Security Best Practices}

\begin{tcolorbox}[warningbox, title={Important Security Guidelines}]
\begin{itemize}[noitemsep, topsep=4pt]
  \item Add \texttt{.env} to \texttt{.gitignore} immediately. \textbf{Never commit credentials to version control.}
  \item In the Google Cloud Console, restrict Firebase API keys to authorised domains only (e.g., your \texttt{web.app} domain).
  \item Rotate Gemini API keys periodically from the Google AI Studio dashboard.
  \item Enable Firebase App Check to block unauthorised clients from accessing your backend.
  \item Review Firestore security rules before every production release.
\end{itemize}
\end{tcolorbox}
